LU2IN005 — TD2 Bounds
3 real TD / annale questions solved step by step

0. Real method for bounds

For one element a:
Above(a) = union of all possible upward paths from a.
Below(a) = union of all possible downward / reverse paths to a.

For a set X = {x1, x2, ...}:
majorants(X) = Above(x1) ∩ Above(x2) ∩ ...
minorants(X) = Below(x1) ∩ Below(x2) ∩ ...

inf(X) = biggest minorant.
sup(X) = smallest majorant.
Order typeTranslate ≤ as...Bounds trick
Product order (a,b) ≤ (c,d) iff a ≤ c and b ≤ d Compare both coordinates.
Sum order (a,b) ≤ (c,d) iff a+b < c+d or same pair Compare sums.
Divisibility order x ≤ y iff x | y minorants = common divisors, majorants = common multiples.

Question 1 — TD2 Exercice 2: Product order

Soit R la relation définie sur l’ensemble
E = {(1,3), (3,1), (3,5), (5,3), (5,7), (7,5), (7,7)}
par :

(m1,m2) R (n1,n2) si et seulement si m1 ≤ n1 et m2 ≤ n2.

Pour l’ensemble
A = {(3,5), (5,3), (5,7), (7,5)}
donner, s’ils existent, les éléments maximaux, les éléments minimaux,
les majorants, les minorants, le plus grand élément, le plus petit élément,
la borne inférieure et la borne supérieure.

Step 0 — translate the relation

(m1,m2) ≤ (n1,n2)
means:
m1 ≤ n1 AND m2 ≤ n2

Both coordinates must work.

Step 1 — find minorants(A)

Minorant = below every element of A.

A = {(3,5), (5,3), (5,7), (7,5)}

Test (1,3):
(1,3) ≤ (3,5) yes, because 1≤3 and 3≤5
(1,3) ≤ (5,3) yes, because 1≤5 and 3≤3
(1,3) ≤ (5,7) yes
(1,3) ≤ (7,5) yes

Test (3,1):
(3,1) ≤ (3,5) yes
(3,1) ≤ (5,3) yes
(3,1) ≤ (5,7) yes
(3,1) ≤ (7,5) yes

minorants(A) = {(1,3), (3,1)}

Step 2 — find majorants(A)

Majorant = above every element of A.

Only (7,7) is above all:

(3,5) ≤ (7,7)
(5,3) ≤ (7,7)
(5,7) ≤ (7,7)
(7,5) ≤ (7,7)

majorants(A) = {(7,7)}

Step 3 — inf and sup

inf(A):
We need the greatest minorant.

minorants(A) = {(1,3), (3,1)}

But:
(1,3) ≤ (3,1) is false because 3≤1 is false.
(3,1) ≤ (1,3) is false because 3≤1 is false.

So the two minorants are not comparable.
Therefore inf(A) does not exist.

sup(A):
majorants(A) = {(7,7)}
Only one majorant, so it is the smallest majorant.
sup(A) = (7,7)

Step 4 — minimal/maximal and min/max

éléments minimaux:
(3,5) and (5,3) have nothing below them inside A.
So minimaux = {(3,5), (5,3)}

éléments maximaux:
(5,7) and (7,5) have nothing above them inside A.
So maximaux = {(5,7), (7,5)}

plus petit élément:
does not exist because there are two minimaux not comparable.

plus grand élément:
does not exist because there are two maximaux not comparable.
Final answer: 
éléments minimaux = {(3,5), (5,3)}
éléments maximaux = {(5,7), (7,5)}
minorants(A) = {(1,3), (3,1)}
majorants(A) = {(7,7)}
plus petit élément = n’existe pas
plus grand élément = n’existe pas
borne inférieure = n’existe pas
borne supérieure = (7,7)

Question 2 — TD2 Exercice 3: Order by sum

On définit la relation ≼ sur IN × IN par
(a,b) ≼ (c,d) ssi a+b < c+d ou (a,b) = (c,d).

Soit l’ensemble
A = {(0,2), (2,1), (1,2), (2,2)}.

Déterminer, s’ils existent, les minorants, les majorants,
les éléments minimaux, les éléments maximaux, la borne inférieure,
la borne supérieure, le minimum, le maximum de A.

Step 0 — translate the relation

(a,b) ≤ (c,d)
means:
a+b is smaller than c+d
OR
the two pairs are exactly equal.

So here we compare sums.

Step 1 — calculate sums of A

(0,2) → 0+2 = 2
(2,1) → 2+1 = 3
(1,2) → 1+2 = 3
(2,2) → 2+2 = 4

Step 2 — minorants(A)

A has lowest sum = 2, at (0,2).

A minorant must be below all elements of A.
So it must be below (0,2).

Everything with sum < 2 is below (0,2).
Also (0,2) is below itself by equality.

In N×N:
sum 0: (0,0)
sum 1: (1,0), (0,1)
same pair: (0,2)

minorants(A) = {(0,0), (1,0), (0,1), (0,2)}

Step 3 — majorants(A)

A has highest sum = 4, at (2,2).

A majorant must be above all elements of A.
So it must be above (2,2).

It can be:
- exactly (2,2), or
- any pair (p,q) with p+q > 4.

majorants(A) = {(2,2)} ∪ {(p,q) ∈ N×N | p+q > 4}

Step 4 — inf, sup, min, max

inf(A) = biggest minorant = (0,2)
sup(A) = smallest majorant = (2,2)

minimum = (0,2), because it is inside A and below all A.
maximum = (2,2), because it is inside A and above all A.

éléments minimaux = {(0,2)}
éléments maximaux = {(2,2)}
Final answer: 
minorants(A) = {(0,0), (1,0), (0,1), (0,2)}
majorants(A) = {(2,2)} ∪ {(p,q) ∈ N×N | p+q > 4}
éléments minimaux = {(0,2)}
éléments maximaux = {(2,2)}
borne inférieure = (0,2)
borne supérieure = (2,2)
minimum = (0,2)
maximum = (2,2)

Question 3 — Annale / Examen 2023: Divisibility order

Soit E = N \ {0,1}, muni de la relation d’ordre | définie par
n | m si et seulement si n est un diviseur de m.

Soit A = {2,3,4,5,6,7,8,9}.

Donner l’ensemble des majorants et l’ensemble des minorants de A dans E.
Donner, s’ils existent, la borne supérieure, la borne inférieure,
le plus petit élément, le plus grand élément,
les éléments maximaux et les éléments minimaux.

Step 0 — translate the relation

n | m means:
n divides m.

So:
lower = divisor
upper = multiple

minorants(A) = common divisors of all elements of A, inside E.
majorants(A) = common multiples of all elements of A, inside E.
Careful:
E = N \ {0,1}
So 1 is NOT allowed.

Step 1 — minorants(A)

A = {2,3,4,5,6,7,8,9}

A common divisor of all these numbers is only 1.

But 1 ∉ E.

So:
minorants(A) = ∅

Step 2 — inf(A)

inf(A) = biggest minorant.

But minorants(A) = ∅.

So:
borne inférieure = n’existe pas.

Step 3 — majorants(A)

Majorants = common multiples of:
2,3,4,5,6,7,8,9

We need ppcm(2,3,4,5,6,7,8,9).

Take biggest prime powers:
8 = 2^3
9 = 3^2
5 = 5
7 = 7

ppcm = 2^3 × 3^2 × 5 × 7
     = 8 × 9 × 5 × 7
     = 2520

majorants(A) = multiples de 2520
= {2520, 5040, 7560, ...}

Step 4 — sup(A)

sup(A) = smallest majorant.

The smallest common multiple is 2520.

So:
borne supérieure = 2520.

Step 5 — plus petit / plus grand élément

plus petit élément:
must be inside A and divide all elements of A.

Only 1 divides everything, but 1 is not inside A.
So:
plus petit élément = n’existe pas.

plus grand élément:
must be inside A and be divisible by all elements of A.

The first common multiple is 2520, but 2520 is not inside A.
So:
plus grand élément = n’existe pas.

Step 6 — éléments minimaux

Minimal = inside A, and no smaller element of A divides it.

2: minimal
3: minimal
4: not minimal because 2 divides 4
5: minimal
6: not minimal because 2 divides 6 and 3 divides 6
7: minimal
8: not minimal because 2 divides 8
9: not minimal because 3 divides 9

éléments minimaux = {2,3,5,7}

Step 7 — éléments maximaux

Maximal = inside A, and no bigger element of A is a multiple of it.

2 divides 4,6,8 → not maximal
3 divides 6,9 → not maximal
4 divides 8 → not maximal
5 divides no other element of A → maximal
6 divides no other element of A → maximal
7 divides no other element of A → maximal
8 divides no other element of A → maximal
9 divides no other element of A → maximal

éléments maximaux = {5,6,7,8,9}
Final answer: 
minorants(A) = ∅
majorants(A) = {2520, 5040, 7560, ...}
borne inférieure = n’existe pas
borne supérieure = 2520
plus petit élément = n’existe pas
plus grand élément = n’existe pas
éléments minimaux = {2,3,5,7}
éléments maximaux = {5,6,7,8,9}

Final memory sheet

For one element:
Above(a) = union of all possible upward paths from a.
Below(a) = union of all possible downward/reverse paths to a.

For a set X:
majorants(X) = intersection of Above(x) for all x in X.
minorants(X) = intersection of Below(x) for all x in X.

inf(X) = biggest minorant.
sup(X) = smallest majorant.

min(X) / max(X) must be inside X.
minorants / majorants / inf / sup can be outside X, but must be inside E.

Divisibility:
minorants = common divisors inside E.
majorants = common multiples inside E.
inf = pgcd if allowed.
sup = ppcm if allowed.

Product order:
both coordinates must work.

Sum order:
compare a+b.
